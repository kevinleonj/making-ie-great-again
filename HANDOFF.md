# Handoff

## Last Change: TTS Backend Switch from F5-TTS to Qwen3-TTS 1.7B

### What Changed
**TTS Backend (F5-TTS to Qwen3-TTS 1.7B):**
- **Rewrote** `scripts/upload_ref_audio.py`: Now a two-step pipeline per leader: (1) upload WAV to fal CDN, (2) call `fal-ai/qwen-3-tts/clone-voice/1.7b` with audio URL + transcript to get a speaker embedding. Reads transcripts from `data/trump_transcript.txt` and `data/maduro_transcript.txt`. Saves output to `data/fal_voice_config.json` (replaces old `fal_audio_urls.json`).
- **Rewrote** `backend/services/tts_service.py`: Replaced F5-TTS with Qwen3-TTS. Changed `_load_ref_audio_urls` to `_load_voice_config` (reads from `fal_voice_config.json`). Removed `_load_transcripts` since reference text now comes from voice config. Updated `_generate_real` to call `fal-ai/qwen-3-tts/text-to-speech/1.7b` with `text`, `language`, `speaker_voice_embedding_file_url`, and `reference_text`. Language values: "English" (Trump), "Spanish" (Maduro). TTSResult, _generate_mock, resolve_audio_path, get_tts_service singleton, and mock mode are unchanged.
- **Updated** `backend/config.py`: Changed `fal_tts_model` default from `"fal-ai/f5-tts"` to `"fal-ai/qwen-3-tts/text-to-speech/1.7b"`.
- **Updated** `tests/unit/test_tts_service.py`: Updated model name in constructor calls. Added `fal_voice_config.json` creation to test setup. All mock tests pass.
- **Updated** documentation: `CLAUDE.md`, `README.md`, `HANDOFF.md`, `frontend/pages/architecture.py` all changed from F5-TTS to Qwen3-TTS.
